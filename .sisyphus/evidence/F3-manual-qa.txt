STEP 9: FORCE_COLOR=1 cargo run --example init_harness first_init | grep -c $'\x1b'
STATUS: PASS (after fix)
INITIAL OUTPUT: 0 (FAIL - anstream was stripping ANSI codes)
FIX APPLIED:
  - Root cause: anstream::stderr() was stripping ANSI escape codes independently
    of our ColorMode logic. Even when ColorMode::Auto correctly returned true for
    FORCE_COLOR=1, anstream's AutoStream detected non-TTY output and stripped
    the codes from the byte stream.
  - Fix: Removed anstream dependency and changed ConsoleLayer::on_event to write
    directly to std::io::stderr().lock() instead of anstream::stderr(). Since our
    formatting logic already conditionally includes/excludes ANSI codes based on
    ColorMode, the anstream abstraction was redundant and actively harmful.
  - Files changed:
    - src/log/console.rs: Changed io::Write import, replaced anstream::stderr()
      with std::io::stderr().lock()
    - Cargo.toml: Removed anstream = "0.6" dependency
  - Post-fix output: 4 ANSI sequences detected (grep -o '\x1b\[[0-9;]*m' | wc -l)
ASSERTION: ANSI escape codes present when FORCE_COLOR=1 -> CONFIRMED
